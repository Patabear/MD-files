첫번째
N과 M을 따로 생각하지말고
하나의 자료구조에 저장해서 계산해야함

해킹에 시도된 비밀번호는 따질필요가 없음
(p1, p2, p3, ...)

그러므로 해킹된 비밀번호로부터 시작해서 찾기

아이디어는 N중의 숫자 하나로 M개의 비밀번호랑 비교해서 따지는게 아니라

M개의 비밀번호중 하나가 자기랑 안전거리가 1인애들을 쭉 찍음
이렇게 안전거리가 1인애들은 이미 끝남 더 찾을필요가없음
왜냐 다른해커비밀번호랑 길이가 어떻든간에 이미 1이라 쓸모가 없어졌기때문

ex) p1으로부터 안전거리가 1인애들 2인애들 찾기
p2로부터 안전거리가 1인애들 2인애들 찾기



그래서 비트거리가 1 차이나는애들 찾는방법
어떤 수 3랑
1과 비교 1나옴 그럼 3-1 => 2
2와 비교 1나옴 그럼 3-2 => 1
4와 비교 0나옴 그럼 3+4 => 7
8과 비교 0나옴 그럼 3+8 => 11

1차이나는애들은 2, 1, 7, 11
